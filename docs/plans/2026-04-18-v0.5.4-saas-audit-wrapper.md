# v0.5.4 — SaaS Audit-Wrapper

**Date:** 2026-04-18
**Predecessors:** v0.5.3 (gate mop-up wave 1).
**Scope:** Route every authenticated `sos.services.saas.app` call through the
unified audit spine. Do NOT replace `require_admin` / `require_customer` —
they encode saas-specific auth (master key + tokens.json slug lookup) that
the kernel gate was not designed for.

## Why not `can_execute()`?

v0.5.1's `can_execute()` assumes a bearer token registered in the kernel's
auth store with FMAAP-compatible scopes. `saas` has two orthogonal auth
systems:

1. **Admin** — `MUMEGA_MASTER_KEY` / `SOS_SAAS_ADMIN_KEY` env secret, not a
   token row in `tokens.json`.
2. **Customer** — `sk-{slug}-{hex}` tokens looked up by SHA-256 hash in
   `tokens.json` (`scope=customer`) with a sqlite fallback via
   `bus_token` → tenant slug. The project field in tokens.json is not
   always the short tenant slug.

Forcing these into the kernel gate would either (a) register every master
key and customer token in kernel auth (duplicating the table of truth) or
(b) bypass the gate's first pillar — neither is an improvement over the
status quo. Audit-wrapper gives us the observability win (every route
emits a `POLICY_DECISION` event) without a table mismatch.

## What we want

- One `POLICY_DECISION` audit event per authenticated call (both allow
  and deny).
- `tenant` on the event matches the tenant slug the call is scoped to
  (admin calls default to `mumega`; customer calls use the resolved slug).
- `action` matches `saas:<operation>` — e.g. `saas:tenant_create`,
  `saas:my_dashboard`.
- `policy_tier` = `saas_admin` or `saas_customer` so audit readers can
  distinguish the two populations.

## The pattern

```python
# sos/services/saas/app.py — new helpers

async def _emit_policy(
    *, agent: str, action: str, target: str, tenant: str,
    allowed: bool, reason: str, tier: str,
) -> None:
    """Emit one POLICY_DECISION to the audit spine. Never raises."""
    from sos.kernel.audit import append_event, new_event
    from sos.contracts.audit import AuditDecision, AuditEventKind
    try:
        await append_event(new_event(
            agent=agent,
            tenant=tenant,
            kind=AuditEventKind.POLICY_DECISION,
            action=action,
            target=target,
            decision=AuditDecision.ALLOW if allowed else AuditDecision.DENY,
            reason=reason,
            policy_tier=tier,
        ))
    except Exception as exc:
        log.debug("audit emit failed (non-fatal): %s", exc)


def audited_admin(action: str):
    """Factory: FastAPI dep that enforces admin auth AND audits the decision."""
    async def _dep(credentials: HTTPAuthorizationCredentials | None = Depends(_bearer)) -> None:
        try:
            require_admin(credentials)
        except HTTPException as exc:
            await _emit_policy(
                agent="anonymous", action=action, target="admin", tenant="mumega",
                allowed=False, reason=str(exc.detail), tier="saas_admin",
            )
            raise
        await _emit_policy(
            agent="admin", action=action, target="admin", tenant="mumega",
            allowed=True, reason="master_key", tier="saas_admin",
        )
    return _dep


def audited_customer(action: str):
    """Factory: FastAPI dep that enforces customer auth, audits, returns tenant_slug."""
    async def _dep(credentials: HTTPAuthorizationCredentials | None = Depends(_bearer)) -> str:
        try:
            tenant_slug = require_customer(credentials)
        except HTTPException as exc:
            await _emit_policy(
                agent="anonymous", action=action, target="customer", tenant="unknown",
                allowed=False, reason=str(exc.detail), tier="saas_customer",
            )
            raise
        await _emit_policy(
            agent="customer", action=action, target=tenant_slug, tenant=tenant_slug,
            allowed=True, reason="tenant_token", tier="saas_customer",
        )
        return tenant_slug
    return _dep
```

## Route inventory (40 authenticated)

**Admin (29)** — replace `Depends(require_admin)` with
`Depends(audited_admin("saas:<action>"))`:

- `saas:tenant_create`, `saas:tenants_list`, `saas:tenant_read`,
  `saas:tenant_update`, `saas:tenant_activate`, `saas:tenant_suspend`
- `saas:seat_create`, `saas:seats_list`, `saas:seat_revoke`
- `saas:usage_record`, `saas:usage_read`, `saas:transaction_record`,
  `saas:invoice_read`, `saas:revenue_read`, `saas:audit_read`
- `saas:rate_limit_check`, `saas:audit_tool_call`
- `saas:marketplace_listings_read`, `saas:marketplace_listing_create`,
  `saas:marketplace_subscribe`, `saas:marketplace_subscriptions_read`,
  `saas:marketplace_earnings_read`
- `saas:notifications_set`, `saas:notifications_read`
- `saas:customer_onboard`
- `saas:build_enqueue`, `saas:build_status`
- `saas:domain_set`, `saas:domain_remove`

**Customer (11)** — replace `Depends(require_customer)` with
`Depends(audited_customer("saas:<action>"))`:

- `saas:my_connect`, `saas:my_dashboard`, `saas:my_wallet`,
  `saas:my_transactions`, `saas:my_tasks_list`, `saas:my_task_create`
- `saas:my_squads`, `saas:my_activity`, `saas:my_invite`, `saas:my_chat`

Public / webhook routes (`/`, `/health`, `/auth/*`, `/webhooks/*`,
`/signup`, `/billing/webhook`) are left unchanged — they have their own
signature/IP verification and emit service-specific audit via
`log_admin` / `_audit_log_tool_call` where needed.
