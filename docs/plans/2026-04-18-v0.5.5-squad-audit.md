# v0.5.5 — Squad Audit (in-dep)

**Date:** 2026-04-18
**Predecessor:** v0.5.4 (saas audit-wrapper).
**Scope:** Emit a `POLICY_DECISION` audit event on both allow and deny
paths of `sos/services/squad/auth.py::require_capability` so every one of
squad's 26 authenticated routes lands on the kernel spine — without
forcing the ~~`can_execute()`~~ gate over squad's native capability
model.

## Why not `can_execute()` directly?

`can_execute()` accepts a `capability=` kwarg but treats it as a
pass-through that only activates under `SOS_REQUIRE_CAPABILITIES=1`.
Squad already does proper capability verification via
`sos.kernel.capability.verify_capability(capability, action, resource)`
inside its `require_capability` dependency — a richer, signed check than
the gate currently performs. Re-routing through the gate would either
*weaken* the check (when the env flag is off) or duplicate it (when on).

The clean move: leave `verify_capability` where it is and add one line of
audit emission at the top of the allow path and the deny path.

## The pattern

`require_capability(resource, operation)` is already a closure factory.
We modify the inner `dependency` coroutine to emit `POLICY_DECISION`:

```python
async def dependency(request, credentials = Depends(security)):
    action_name = f"squad:{resource}_{operation}"
    token = credentials.credentials if credentials else ""
    if not token:
        await _emit_squad_policy(
            agent="anonymous", action=action_name, target=resource,
            tenant="unknown", allowed=False, reason="missing_authorization",
        )
        raise HTTPException(401, "missing_authorization")
    auth = _lookup_token(token, database)
    if not auth:
        await _emit_squad_policy(
            agent="anonymous", action=action_name, target=resource,
            tenant="unknown", allowed=False, reason="invalid_token",
        )
        raise HTTPException(401, "invalid_token")
    if auth.is_system:
        await _emit_squad_policy(
            agent=auth.identity.id, action=action_name, target=resource,
            tenant="mumega", allowed=True, reason="system_token",
        )
        return auth
    capability = _capability_for(auth.identity, auth.tenant_id, action)
    ok, reason = verify_capability(capability, action, _resource_name(resource, auth.tenant_id))
    tenant_for_audit = auth.tenant_id or "mumega"
    if not ok:
        await _emit_squad_policy(
            agent=auth.identity.id, action=action_name, target=resource,
            tenant=tenant_for_audit, allowed=False, reason=reason or "capability_denied",
        )
        raise HTTPException(403, reason)
    await _emit_squad_policy(
        agent=auth.identity.id, action=action_name, target=resource,
        tenant=tenant_for_audit, allowed=True, reason="capability_ok",
    )
    return auth
```

`_emit_squad_policy` mirrors the saas helper:

```python
async def _emit_squad_policy(
    *, agent: str, action: str, target: str, tenant: str,
    allowed: bool, reason: str,
) -> None:
    try:
        from sos.kernel.audit import append_event, new_event
        from sos.contracts.audit import AuditDecision, AuditEventKind
        await append_event(new_event(
            agent=agent, tenant=tenant,
            kind=AuditEventKind.POLICY_DECISION,
            action=action, target=target,
            decision=AuditDecision.ALLOW if allowed else AuditDecision.DENY,
            reason=reason, policy_tier="squad_capability",
        ))
    except Exception:
        # Audit must never break a request.
        pass
```
