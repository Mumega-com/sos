# v0.6.2 — Squad schema cleanup

**Date:** 2026-04-18
**Branch:** codex/sos-runtime-validation
**Prior:** v0.6.1 (test debt closed)

## Goal

Drop 6 legacy columns from the squad schema that are defined in
`models.py` + `alembic/versions/0001_initial.py` but have zero
runtime readers/writers.

## Columns removed

| Table            | Column             | Replaced by |
|------------------|--------------------|-------------|
| squad_skills     | `framework`        | `sos/adapters/*` — framework wiring at adapter layer |
| squad_skills     | `agent`            | `squads.members_json` + `roles_json` + dynamic dispatch |
| pipeline_specs   | `review_enabled`   | `sos/kernel/policy/gate.py` — `can_execute()` gate signal |
| pipeline_specs   | `review_agent`     | arbitration (v0.5.2) — chooses reviewer at decide-time |
| pipeline_specs   | `review_cmd`       | gate propose + audit stream (v0.5.6) |
| pipeline_runs    | `reviewer_notes`   | `/audit` stream events tagged with `trace_id` |

## Evidence of safety

Grep across `/sos` found zero references outside:
- `sos/services/squad/models.py` — the defs themselves
- `sos/services/squad/alembic/versions/0001_initial.py` — the baseline migration

The migration's own docstring (lines 12-16) marks them as drop
candidates for v0.6.1+.
