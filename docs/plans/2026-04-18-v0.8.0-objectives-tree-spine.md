# v0.8.0 — Objectives Tree Spine

**Date:** 2026-04-18
**Author:** Codex (with Hadi)
**Thesis:** At 1M users × N agents per user, messaging cannot be the coordination
primitive. What scales is a **shared gradient** that every agent reads locally
and pulls work from. We build that gradient as a **living objective tree** —
nodes with bounty, claim, heartbeat, completion gate — and make it the spine
of the organism. Dispatch, channels, and bounty-fractal all compose out of
this one primitive.

This version is SOS-layer only (platform-agnostic). Mumega's CF-native
implementation (D1 + Durable Objects) is a separate subsequent sprint. That
boundary is explicit: see `feedback_sos_vs_mumega_layer.md`.

---

## What this replaces

| Old shape | New shape (node-valued, not row-valued) |
|---|---|
| "Kasra, do X" via direct bus send | Post a node; subscribers pull it |
| `#build-failures` channel | Subscribe to subtree `/ops/alerts/` or tag `build` |
| Central dispatcher | `GET /objectives/open?capability=X` — agents pull |
| `review_agent` / `review_cmd` / `reviewer_notes` columns (dropped v0.6.2) | Reviewer = whoever claims the `review-needed` child node; notes = the completion artifact |
| Bounty "claimed by one squad forever" | Squad claims, posts children, new squads self-form around children |

**The v0.6.2 column-drop was correct for *column* reasons but wrong because
I hadn't named the tree yet.** This plan names it. `review_*` columns are
not being restored as columns; they're being restored as the *first canonical
objective subtree* (`/reviews/`), which proves the pattern on the primitive
that motivated the callout.

## Steps

### Step 1: Objective contract
**File:** `/mnt/HC_Volume_104325311/SOS/sos/contracts/objective.py`
**Change:** New Pydantic v2 model `Objective`. Fields:
- `id: str` (ULID)
- `parent_id: str | None`
- `title: str`
- `description: str = ""`
- `bounty_mind: int = 0`
- `state: Literal["open", "claimed", "shipped", "blocked"] = "open"`
- `holder_agent: str | None` — set on claim, cleared on release
- `holder_heartbeat_at: str | None` — ISO-8601, bumped every heartbeat
- `subscribers: list[str] = []` — agent IDs
- `tags: list[str] = []`
- `capabilities_required: list[str] = []`
- `created_by: str`
- `created_at: str`
- `updated_at: str`
- `tenant_id: str = "default"`
- `project: str | None = None`
Plus `@classmethod now_iso()`, `to_redis_hash()`, `from_redis_hash()` — mirror
`sos/contracts/agent_card.py` shape exactly so the storage layer can reuse the pattern.
**Outcome:** `pytest tests/contracts/test_objective.py -q` passes 6+ tests covering validation, round-trip, state transitions.

### Step 2: JSON Schema snapshot
**File:** `/mnt/HC_Volume_104325311/SOS/sos/contracts/schemas/objective.schema.json` (new, generated) + `/mnt/HC_Volume_104325311/SOS/tests/contracts/test_objective_schema.py`
**Change:** Dump `Objective.model_json_schema()` to the JSON file; snapshot-test to prevent silent contract drift. Follows pattern from `agent_card.schema.json`.
**Outcome:** Schema file committed; test fails loudly if fields change without regenerating.

### Step 3: Kernel claim/heartbeat helpers
**File:** `/mnt/HC_Volume_104325311/SOS/sos/kernel/objectives.py` + `/mnt/HC_Volume_104325311/SOS/tests/kernel/test_objectives.py`
**Change:** Three fail-soft httpx POST wrappers, mirroring `sos/kernel/heartbeat.py` exactly:
- `claim(objective_id: str, *, base_url=None, token=None, timeout_s=5.0) -> bool`

Reads `SOS_OBJECTIVES_URL` / `SOS_OBJECTIVES_TOKEN` / `SOS_SYSTEM_TOKEN`. Never raises. Non-2xx → False. No token → False.
**Outcome:** 6 tests (no-token, success, claim-already-held-409→False, heartbeat-success, release-success, exception→False) pass.

### Step 4: Service storage layer
**File:** `/mnt/HC_Volume_104325311/SOS/sos/services/objectives/__init__.py` (new module)
**Change:** Redis-backed storage functions, mirroring `sos/services/registry/__init__.py`:
- Key prefix `sos:objectives:{project or "default"}`
- `write_objective(obj, ttl_seconds=None)` — hash-set the object + `SADD` to parent's children set
- `read_objective(id, project=None)`
- `read_children(parent_id, project=None) -> list[Objective]`
- `read_tree(root_id, project=None, max_depth=10) -> dict` — recursive, returns nested dict
- `query_open(project=None, tag=None, min_bounty=None, subtree_root=None, capability=None) -> list[Objective]`
- `claim_objective(id, agent, project=None) -> bool` — Redis transaction; returns False if already claimed
- `heartbeat_objective(id, project=None) -> bool`
- `release_objective(id, project=None)` — also called by decay sweeper
- `complete_objective(id, artifact_url, project=None)` — state→shipped pending ack
- `ack_completion(id, acker_agent, project=None)`
All fail-soft; fakeredis-backed in tests.
**Outcome:** `pytest tests/services/test_objectives_storage.py -q` — 15+ tests cover each function including tree walk, claim-race, decay.

### Step 5: Service app (FastAPI routes)

### Step 7: Audit — emit state transitions to bus
**File:** `/mnt/HC_Volume_104325311/SOS/sos/services/objectives/app.py` (edit same file)

### Step 9: Completion gate + economy wire
**File:** `/mnt/HC_Volume_104325311/SOS/sos/services/objectives/gate.py` + edit `sos/services/objectives/app.py` to call it.
**Change:**
- `check_completion(objective_id, project)` — read obj; if `state == "shipped"` and (parent-holder has acked OR ≥ N peer acks), transition → `paid`; call `AsyncEconomyClient.transfer_bounty(from_treasury, to=holder_agent, amount=obj.bounty_mind, memo=f"obj:{id}")`.
- If ack fails the check → state stays `shipped` (awaiting more acks).
- Default N = 1 peer ack OR parent-holder ack, whichever comes first. Config per-node via `completion_policy` field (v0.8.1 extends; v0.8.0 hardcodes N=1).
**Outcome:** end-to-end test: create obj with bounty → claim → complete (with artifact) → ack → economy transfer called with correct amount → obj.state == `paid`.

## Why this is the right version to ship now

The v0.6.2 callout exposed that I collapse primitives without naming their replacement. The 1M-scale question forced the spine into view: **the objective tree is the gradient, and every other comms primitive (dispatch, channels, bounty-fractal) composes out of it.** Shipping v0.8.0 gives the organism its first real coordination surface that scales past direct messaging. Everything that follows (CF impl, subscriptions, decay, fractal UX) is an elaboration, not a re-think.

The squad is picked. The steps are atomic. The scope is honest. Ready to execute.
