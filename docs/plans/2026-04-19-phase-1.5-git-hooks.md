**Date:** 2026-04-19
**Parent:** `docs/plans/2026-04-19-mumega-mothership.md` (Phase 1.5 between v0.9.0 ports and v0.9.1 bus stability)
**Task:** #214

**End-state:** Every commit — Codex, Kasra, Claude, manual, future agent — passes the same invariants before it can land. Broken ports, lint drift, and contract regressions cannot reach any branch.

---

## Approach (B + C combined)

**Split fast/thorough + all-local hooks:**

- **pre-commit stage** (every `git commit`, target ≤3s): ruff, black, `lint-imports` (import-linter R1–R7), `make contracts-check` (port schema drift).
- **pre-push stage** (every `git push`, target ≤15s): `pytest tests/contracts/ -q`.
- **all-local**: every hook is `language: system`, calling the same binary CI calls. Zero version drift between local and CI.
- **CI mirrors locally**: the lint/import-test/test jobs in `.github/workflows/ci.yml` collapse into a single `pre-commit run --all-files --hook-stage push` step that runs exactly what a developer ran on their machine.

## Current state (from exploration)

### Step 2 — Create `scripts/install-hooks.sh` (one-command bootstrap)
**File:** `/mnt/HC_Volume_104325311/SOS/scripts/install-hooks.sh`
**Change:** New bash script (chmod +x):
```bash
#!/usr/bin/env bash
set -euo pipefail
# Install pre-commit framework if missing.
if ! command -v pre-commit >/dev/null 2>&1; then
  pip install pre-commit>=3.0
fi
pre-commit install --hook-type pre-commit
pre-commit install --hook-type pre-push
echo "hooks installed: pre-commit + pre-push"
echo "run 'pre-commit run --all-files' to test"
```

**Outcome:** `bash scripts/install-hooks.sh` leaves `.git/hooks/pre-commit` and `.git/hooks/pre-push` in place. Idempotent — safe to re-run.

### Step 4 — Collapse `.github/workflows/ci.yml` to mirror local hooks
**File:** `/mnt/HC_Volume_104325311/SOS/.github/workflows/ci.yml`
**Change:** Replace the three separate `lint`, `import-test`, `test` jobs with a single `pre-commit` job:
```yaml
pre-commit:
  name: Hooks (lint + contracts + tests)
  runs-on: ubuntu-latest
  steps:
    - uses: actions/checkout@v4
    - uses: actions/setup-python@v5
      with: { python-version: "3.11" }
    - name: Install project (for import-linter + pytest)
      run: pip install -e ".[dev]"
    - name: Run pre-commit hooks (commit + push stages)
      run: |
        pre-commit run --all-files --hook-stage commit
        pre-commit run --all-files --hook-stage push
```
Keep the `deploy` job's `needs: [pre-commit]`.
**Outcome:** CI runs the literal-same commands a developer runs. No drift surface. Pre-existing `import-test`'s hand-written import-smoketest is dropped — import-linter subsumes it.

## Rollback

If hooks block legitimate work:
1. Dev can bypass a single commit with `git commit --no-verify` (documented in `install-hooks.sh` stdout).
2. Revert the commit — hooks uninstall themselves next `pre-commit uninstall`.
3. CI still catches what local skipped (belt+suspenders).
