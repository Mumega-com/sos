## Goal

Restore clean `.venv/bin/python -m pytest tests/` green state by fixing the six pre-existing 403 Forbidden failures across `tests/services/test_settlement.py` and `tests/services/test_economy_usage.py`. These regressed when the v0.5.x policy-gate / audit-wrapper work landed on the economy service.

## Constraints (from Hadi, 2026-04-18)

- Stateful squad anchor: `sos-connectivity-medic` (Sonnet, Claude Code project agent).

## Current state

Failures (all 403 where 200/201/404 expected — one shared root cause):

- `test_settlement::test_retry_deferred_works_when_funds_arrive`
- `test_settlement::test_settle_endpoint_404_for_unknown_id`
- `test_economy_usage::test_system_token_can_write_any_tenant`
- `test_economy_usage::test_imagen_4_event_roundtrip`
- `test_economy_usage::test_tenant_sees_own_events_only`
- `test_economy_usage::test_system_token_sees_all_if_no_filter`

Fixture context: both test files monkeypatch `sos.kernel.auth.TOKENS_PATH` to a tmp-dir tokens file containing an `admin` and `trop` token. That fixture predates the v0.5.1 policy gate + v0.5.5/5.6 audit-wrapper changes, which now reject these routes despite valid tokens.
