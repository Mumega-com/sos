# SOS Structural Audit — 2026-04-17

Authoritative snapshot of architectural boundary violations in the SOS repo,
generated by a research-only audit against the target micro-kernel model.

## Target boundary model

```
sos/
  kernel/        minimal core — identity, tokens, auth, bus primitives
  contracts/     ABI — JSON Schemas + Pydantic + OpenAPI (shared by both sides)
  services/      one process per service; never imports another service
  clients/       HTTP SDK per service; imports contracts only
  cli/           mumega end-user chat app
  adapters/      outbound integrations (Discord, Telegram, CrewAI, ToRivers)
  mcp/           MCP protocol bridge (thin proxy)
  agents/        per-agent genesis / seed files
  bus/           bus config (tokens.json) — data, not code
  skills/        skill registry content
  deprecated/    retired code, walled off
```

## Rules

- **R1** — `sos/services/<A>/` never imports `sos/services/<B>/` (A ≠ B). Crossing goes through `sos/contracts/` or runtime IPC (bus / Redis / HTTP).
- **R2** — `sos/clients/`, `sos/adapters/`, `sos/mcp/`, `sos/cli/`, `sos/agents/` may import `sos/contracts/` but never `sos/services/<X>/` internals.
- **R3** — Bus producers and HTTP handlers build messages/responses via Pydantic models in `sos/contracts/*` — never hand-rolled dicts.
- **R4** — `scripts/` holds only one-off admin/ops utilities. Product flows live with their service.
- **R5** — `sos/deprecated/` and `.archive/` are walled off; no live imports.
- **R6** — `tests/contracts/` imports only `sos.contracts.*`. Cross-service integration tests live in `tests/integration/`.

## Services that are actually shared infrastructure

These live under `services/` but function as libraries — six+ services import them:

- `services/bus/core.py` → **`kernel/bus.py`**
- `services/auth/` → **`kernel/auth.py`**
- `services/_health.py` → **`kernel/health.py`**
- `services/common/` → **`kernel/policy/`** (stub for v0.5.0 policy engine)

## What's clean

- `sos/kernel/` — no violations detected
- `sos/contracts/` — no violations; missing `Tenant` model is the main gap
- `sos/deprecated/` — walled off correctly; no live imports
- `sos/observability/`, `sos/plugins/`, `sos/providers/` — clean
- Bus envelope enforcement (`enforcement.py::_V1_TYPES`) — working immune system
