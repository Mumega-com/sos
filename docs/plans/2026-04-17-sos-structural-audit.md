# SOS Structural Audit ‚Äî 2026-04-17

Authoritative snapshot of architectural boundary violations in the SOS repo,
generated by a research-only audit against the target micro-kernel model.

## Target boundary model

```
sos/
  kernel/        minimal core ‚Äî identity, tokens, auth, bus primitives
  contracts/     ABI ‚Äî JSON Schemas + Pydantic + OpenAPI (shared by both sides)
  services/      one process per service; never imports another service
  clients/       HTTP SDK per service; imports contracts only
  cli/           mumega end-user chat app
  adapters/      outbound integrations (Discord, Telegram, CrewAI, ToRivers)
  mcp/           MCP protocol bridge (thin proxy)
  agents/        per-agent genesis / seed files
  bus/           bus config (tokens.json) ‚Äî data, not code
  skills/        skill registry content
  deprecated/    retired code, walled off
```

## Rules

- **R1** ‚Äî `sos/services/<A>/` never imports `sos/services/<B>/` (A ‚âÝ B). Crossing goes through `sos/contracts/` or runtime IPC (bus / Redis / HTTP).
- **R2** ‚Äî `sos/clients/`, `sos/adapters/`, `sos/mcp/`, `sos/cli/`, `sos/agents/` may import `sos/contracts/` but never `sos/services/<X>/` internals.
- **R3** ‚Äî Bus producers and HTTP handlers build messages/responses via Pydantic models in `sos/contracts/*` ‚Äî never hand-rolled dicts.
- **R4** ‚Äî `scripts/` holds only one-off admin/ops utilities. Product flows live with their service.
- **R5** ‚Äî `sos/deprecated/` and `.archive/` are walled off; no live imports.
- **R6** ‚Äî `tests/contracts/` imports only `sos.contracts.*`. Cross-service integration tests live in `tests/integration/`.
