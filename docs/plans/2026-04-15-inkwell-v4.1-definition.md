# Inkwell 4.1

## One sentence

**Your business, running itself, showing you everything.**

---

## The idea

Inkwell 4.0 gave businesses a digital operating system. Contracts, payments, dashboards, courses, discovery — all from one config file.

Inkwell 4.1 makes it alive.

When a squad operates your business, you shouldn't have to ask "what happened today?" You open your site. Every decision is there. Every metric is charted. Every dollar is tracked. Every goal has a progress bar. The system documents itself as it works.

And when you want to sell knowledge — a book, a course, a premium report — you don't need another platform. You write it. You set the price. Inkwell handles the rest: payments, access, reader accounts, progress tracking, release notifications.

---

## What changes

### The Glass

Every squad gets a self-writing dashboard. Not a dashboard you build — a dashboard that builds itself from the squad's activity.

- **Metrics** come from the flywheel (GSC, GA4, ad spend). Updated daily. Charted automatically.
- **Transactions** come from the wallet. Every earn, spend, and transfer logged with reason.
- **Goals** come from the squad's objectives. Progress bars that move when tasks complete.
- **Decisions** come from the event log. A timeline of what happened and why.
- **Health** comes from the heartbeat. Green, yellow, red. Updated every 5 minutes.

No LLM generates these pages. SQL query → template → static page. Deterministic.

### The Gate

Every action passes through FMAAP before it happens. Five questions:

1. Is the system healthy? (Flow)
2. Can we afford this? (Metabolism)
3. Is this what we do? (Alignment)
4. Is this agent allowed? (Autonomy)
5. Is the physics stable? (Coherence)

If any answer is no, the action doesn't happen. Not logged as a failure — logged as a gate. The system protects itself.

### The Shelf

Books, courses, reports, premium content — sold natively. No Substack. No Gumroad. No GHL.

- Content lives in the CMS (Shabrang or any markdown source)
- Commerce lives in Inkwell (Stripe, access grants, reader library)
- The reader sees: landing page → checkout → library → reading progress
- The creator sees: sales on the dashboard, readers on the metrics page

One fork can sell vehicle shipping contracts. Another fork can sell AI governance courses. Another can sell serialized fiction. Same engine. Different config.

### The Routing

Tasks find the right squad automatically. No dispatcher. No assignment meetings.

```
dG/dt = |F|^γ - αG
```

Squads that do good work on a skill get more of that work. Squads that stop doing a skill lose conductance. The system routes itself.

## What it looks like

```
Customer opens their Inkwell site:

/                   → their business homepage
/dashboard          → live KPIs, auto-updated daily
/dashboard/goals    → progress bars on quarterly objectives
/dashboard/costs    → donut chart of spend by category
/dashboard/health   → green badges, last heartbeat 30s ago
/library            → their published books/courses
/portal             → customer-facing contract signing
/discovery          → business assessment quiz
```

They didn't build any of this. Their squad did. And the squad didn't build it either — it wrote itself from the data.
