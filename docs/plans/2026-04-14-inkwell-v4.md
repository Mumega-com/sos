# Plan: Inkwell v4 — AI-First Digital Marketing Platform

**Date:** 2026-04-14
**Decision:** Build on existing Mumega-com/inkwell repo, v4 branch
**Architecture:** Modified Option C (Codex) — one Worker, split D1, scheduled ingestion, SSR shell + client islands
**First customer:** Viamar (viamar.ca)

## Vision

Each customer gets a fork of Inkwell that IS their digital marketing platform. Not a tenant — their own sovereign instance.

```
Inkwell v4 = Content Engine (Astro 6, markdown, Zod)
           + Data Backend (Cloudflare Workers, Hono, D1)
           + Connectors (GSC, GA4, Ads, GHL, MumCP, Notion)
           + Dashboard (SSR shell + React chart islands)
           + Customer Portal (auth, estimator, tracking)
           + Agent Interface (agent operates via config + markdown)
```

## Storage (Cloudflare)

| Binding | Type | Purpose |
|---------|------|---------|
| DB_CORE | D1 | Users, sessions, quotes, shipments, checklists |
| DB_MARKETING | D1 | GSC/GA4/Ads snapshots, rollups, weekly deltas |
| SESSIONS | KV | Session tokens (TTL 30d), magic link codes (TTL 5min) |
| CONTENT | KV | Notion-synced content cache |
| MEDIA | R2 | Images downloaded from Notion, uploaded by users |

## Notion Sync Flow

```
Customer edits in Notion (familiar UI, images, rich text)
    ↓
Scheduled Worker (every 5 min)
    ↓
1. Notion API → fetch pages where Status = "Published"
2. Convert blocks → markdown + frontmatter
3. Download images → R2, rewrite URLs
4. Store in KV (CONTENT namespace)
5. POST deploy hook → Cloudflare Pages rebuilds
    ↓
Site live with new content (< 2 min from Notion publish)
```

## Auth Flow (Magic Link)

```
1. User enters phone/email on /portal/login
2. Worker: generate 6-digit code → D1 magic_codes + send via GHL SMS
3. User enters code
4. Worker: validate → create session in KV → set cookie
5. Redirect to /dashboard or /portal
6. All /dashboard/* and /portal/* routes check session middleware
```

## Connector Plugin Architecture

Each connector follows the same interface:

```typescript
interface Connector {
  name: string
  fetch(config: ConnectorConfig): Promise<Snapshot[]>
  normalize(raw: any): NormalizedMetric[]
}

interface NormalizedMetric {
  source: string       // "gsc" | "ga4" | "google-ads" | "ghl"
  metric: string       // "clicks" | "impressions" | "leads" | "spend"
  value: number
  dimensions: Record<string, string>  // { query: "...", page: "..." }
  period: string       // "2026-04-14"
}
```

New customer needs HubSpot instead of GHL? Write `hubspot.ts` implementing the same interface. No Worker changes.
