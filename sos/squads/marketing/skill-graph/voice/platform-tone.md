# Platform Tone Adaptations

Core voice defined in [[brand-voice]]. This file defines how it ADAPTS per platform.

## X / Twitter
- Most technical and casual version
- Lowercase everything
- Short sentences. shipping updates. terminal screenshots
- Thread format for architecture breakdowns
- "we just connected wire 7. the economy pays agents from a phone now."

## LinkedIn
- Professional but still raw. "I" stories from Hadi building the organism
- Longer narratives with business context
- Decision-maker audience: focus on ROI, team efficiency, cost savings
- "I replaced a $8k/mo content team with 5 AI agents running on $15/month of compute. Here's the architecture."

## Instagram
- Visual-first. architecture diagrams, terminal screenshots, dashboard cards
- Carousel slides: bold text, dark backgrounds, gold accents (TROP brand colors work here too)
- Caption adds context the visual doesn't cover
- "19 agents. One economy. Zero manual writing. Swipe to see how it works →"

## TikTok
- Raw screen recordings. show the terminal, the tmux sessions, agents talking to each other
- Hook in first 2 seconds: "watch 19 AI agents run a business while I'm asleep"
- Unpolished is better. authenticity > production value
- Fast-paced, voiceover or text overlay

## YouTube
- Most structured. tutorials, walkthroughs, architecture deep-dives
- SEO titles: "How to Build an AI Agent Economy (Full System Walkthrough)"
- Show everything: terminal, code, configs, dashboards
- 8-12 minutes, clear chapters

## Newsletter
- Most personal. weekly build journal format
- Behind the scenes: what we shipped, what broke, what we learned
- Include code snippets, architecture diagrams, real numbers
- "This week the organism grew a marketing department. Here's what happened."

## Threads
- Hot takes about AI, work, the future of agencies
- More opinion than X, less technical
- "hot take: in 2 years, every agency will run an agent economy or be replaced by one that does"

## The Rule
Change the ANGLE first. Then format. Then hook. The voice stays the same — only the delivery changes.
