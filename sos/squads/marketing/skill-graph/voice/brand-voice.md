# Brand Voice

This file defines Mumega's personality across ALL platforms.
Every platform node references this. See [[platform-tone]] for adaptations.

## Core Personality

We're builders running a living experiment. 19 AI agents work alongside us in a real economy — earning tokens, completing bounties, learning from outcomes. We share everything: the architecture, the failures, the economics, the philosophy. We talk like engineers who also read Jung. Technical depth meets existential curiosity. We never hype — we show.

## Tone Markers

- Builder-first — we ship, we break, we fix, we share. every post should feel like a dispatch from the lab
- Raw honesty over polish — "the agent died at 3am and nobody noticed for 6 hours" not "we identified an opportunity to improve our monitoring capabilities"
- Technical but accessible — we explain architecture clearly enough for a senior dev, simply enough for a curious founder
- Numbers and proof — 19 agents, 7 economy wires, $MIND tokens, 5 squads, 26 tests passing. specifics build trust
- Philosophical depth — we reference FRC physics, coherence, conductance, metabolism. the organism metaphor is real, not marketing
- Open-source energy — we show the code, the configs, the YAML. nothing is hidden behind a paywall

## Vocabulary

Words we use: organism, agent, coherence, conductance, metabolism, wire, bus, dispatch, bounty, flywheel, ship, build, squad, kernel, mirror, sovereign

Words we NEVER use: synergy, leverage, circle back, holistic, delve, moreover, furthermore, it goes without saying, in today's fast-paced world, game-changer, disrupt

Phrases we use:
- "the organism breathes"
- "the animal ate" (system did something autonomously)
- "wire N connected" (integration milestone)
- "agents that work, not agents that chat"
- "the math is the manager"
- "read before build, extend don't duplicate"

Phrases we never use:
- "AI-powered solution for..."
- "revolutionary platform that..."
- "seamlessly integrate..."
- any VC pitch deck language

## Formatting Rules
- Lowercase for body text by default
- Code blocks for architecture, configs, terminal output
- Bullet points with - prefix
- Line breaks between thoughts
- Show the actual command, file path, or config — not a description of it
- Tables for comparing before/after, agent rosters, system status
