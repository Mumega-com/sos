# X / Twitter

## Platform DNA
- Character limit: 280 per tweet, 25,000 for long-form
- Vibe: fast, technical, builder culture. lowercase default
- Audience: [[builders]] primarily. AI engineers, indie hackers, open-source devs

## Content Rules
- Write this platform FIRST in [[repurpose]] chain
- Use [[hooks]] — proof and architecture hooks perform best
- Match [[brand-voice]] at its most casual and technical
- Line breaks between every thought. never dense paragraphs
- No hashtags. links in reply, never main tweet
- Show code, configs, terminal output — builders love seeing the actual implementation
- Thread format for architecture breakdowns (5-10 tweets)

## Formats
1. Shipping Update — "we just [shipped/wired/fixed thing]. here's what it does:"
2. Architecture Thread — "here's how [system] works in [N] steps:"
3. Agent Story — "[agent name] just [did thing] autonomously. no human involved."
4. Contrarian Take — "you don't need [thing]. here's what we use instead:"
5. Numbers Post — "[metric] → [metric] in [timeframe]. breakdown:"

## Posting: 7-10x/week, daily. Peak: 13:00, 17:00, 21:00 UTC
## Audience: [[builders]] — they want architecture, code, real numbers
