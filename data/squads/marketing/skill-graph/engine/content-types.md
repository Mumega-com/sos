# Content Types & Formats

## Build Journal
- Platforms: blog (Inkwell), newsletter, LinkedIn
- When: after shipping something, weekly recap
- Structure: what we built → how → what broke → what we learned
- Example: "We Wired a Complete Economy in One Session"

## Architecture Thread
- Platforms: X (thread), YouTube
- When: explaining how a system works
- Structure: problem → solution → diagram → code → result
- Example: "here's how 19 agents coordinate on one Redis bus"

## Agent Story
- Platforms: X, LinkedIn, blog
- When: an agent does something notable
- Structure: what happened → why it matters → what it means
- Example: "An AI Agent Fixed 11 System Bugs Without a Human"

## Terminal Demo
- Platforms: TikTok, YouTube Shorts, Instagram Reels
- When: showing the system working live
- Structure: hook (2s) → show terminal → explain → result
- Example: screen recording of agents passing messages

## Carousel / System Diagram
- Platforms: Instagram, LinkedIn
- When: explaining architecture, comparing before/after
- Structure: slide 1 = bold claim, slides 2-8 = system, last = CTA
- Example: "19 Agents. One Economy. Zero Manual Writing."

## Tool Launch
- Platforms: X, ProductHunt, blog, LinkedIn
- When: releasing open-source tools (mumcp, Inkwell, SOS)
- Structure: what it does → demo → how to use → link
- Example: "mumcp: 239 MCP tools for WordPress"

## Hot Take
- Platforms: X, Threads
- When: opinion on AI industry, future of work
- Structure: bold claim + 1-2 lines of reasoning
- Example: "frameworks are where agent architectures go to die"

## Weekly Newsletter
- Platforms: email (Resend/Inkwell)
- When: every Thursday
- Structure: 3 things we shipped + 1 thing that broke + 1 thing we learned + 1 link
- Length: 800-1,500 words
